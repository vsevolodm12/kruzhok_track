## Задание

Нужно построить Telegram Mini App для онлайн школы по олимпиадам по истории. Дизайн придумай сам — должен быть удобным, функциональным, не перегруженным. Приоритетно белая тема, но с возможностью переключаться на тёмную.

**Логотип:** На главной странице должен быть логотип (название "Кружок"). Создай SVG с этим логотипом и органично размести его.

---

## Правила дизайна

**ИЗБЕГАТЬ базовых ИИ-цветов:**
- ❌ Чистый синий (#3B82F6), фиолетовый (#8B5CF6), зелёный (#10B981), красный (#EF4444)
- ❌ Стандартные Material Design / Tailwind default цвета
- Эти цвета сразу выдают "сделано нейросетью"

### Запрет: Цветовое кодирование категорий

**НЕ использовать** разные цвета для разных типов работ:
- ❌ "ДЗ = зелёный, Эссе = синий, Пробник = фиолетовый"
- ❌ Цветные бейджи/теги с типом работы
- ❌ Аватарки-заглушки (буква в цветном круге) — у нас НЕТ фото пользователя

**Как показывать тип работы:**
- Текстом (мелким, приглушённым) 
- Через группировку по секциям
- Иконкой без цветного фона

### Структура

- Контейнеры с фоном — только для основных блоков (карточка статистики, секция)
- Элементы списка разделять отступами или тонкими линиями, а не индивидуальными контейнерами
- Визуальная иерархия через типографику (размер, вес), а не через цвет

# Функционал дашборда ученика

  

**Тип приложения:** Telegram Mini App

**Ориентация:** Вертикальная (portrait) для мобильных устройств

**Тип интерфейса:** SPA-подобный (Single Page Application) — обновление UI без перезагрузок страницы, плавные переходы между вкладками и состояниями

  

---

  

## Источники данных

  

Все данные поступают из вебхуков ZenClass. Ключевой вебхук — `lesson_task_accepted`, который содержит:

  

| Поле webhook | Что содержит | Как используется |

|--------------|--------------|------------------|

| `user_email` | Email ученика | Идентификация пользователя |

| `course_name` | Название курса | Отображение и группировка |

| `task_name` | Название задания | Отображение в списках и деталях |

| `task_type` | Тип задания | Категоризация (эссе/проект/пробник/дз) |

| `comment` | Комментарий преподавателя - только парсинг оценки / балла

| `task_result` | Результат проверки | "ok" или формат "5/7" для тестов |

| `report_link` | Ссылка на отчёт | (опционально) переход к полному отчёту |

| `timestamp` | Unix-время проверки | Дата, сортировка, календарь активности |

| `tarif_name` | Название тарифа | (опционально) разделение по тарифам |

  

---

  

## Сущности данных

  

### Ученик (Student)

- Имя

- Email

- Список доступных курсов

  

### Курс (Course)

- Название курса

- Связь с учеником (у одного ученика может быть несколько курсов)

  

### Работа (Grade)

- Название работы (из `task_name`)

- Тип работы (определяется по `task_type` или паттерну в названии)

- Оценка (парсится из `comment`)

- Максимальный балл (зависит от типа)

- Дата проверки

- Привязка к курсу

  

### Типы работ и система оценивания

  

| Тип | Макс. балл | Особенности |

|-----|-----------|-------------|

| **Эссе** | 50 | Фиксированный максимум |

| **Проект** | 50 | Фиксированный максимум |

| **Пробник** | 100 | Имитация экзамена |

| **Домашнее задание** | Переменный | Максимум зависит от конкретного задания (1/10) |

## Навигация

  

### Три основные вкладки:

1. **История** — хронологический список всех работ

2. **Главная** — общий прогресс и сводка по направлениям

3. **Статистика** — детальные графики и аналитика

  

### Переключение между вкладками

- Анимированный индикатор показывает текущую вкладку

- Плавный переход между страницами

- Состояние сохраняется при переключении

  

---

  

## Страница: Главная (по середине в панеле)

  

### Блок: Информация о пользователе

**Данные:** ТОЛЬКО имя ученика и email. Всё.

**Аватарки НЕТ** — не нужно создавать заглушку (букву в круге и т.п.). У нас нет фото пользователя в данных, не имитируй его.

**Источник:** профиль из БД, изначально из `user_email` вебхука

  

### Блок: Выбор курса

**Функционал:**

- Отображает название текущего курса

- При нажатии — выпадающий список всех доступных курсов ученика

- Переключение курса обновляет все данные на странице

- Текущий курс выделен визуально

  

**Связи:**

- Влияет на ВСЕ отображаемые данные (оценки, статистика, история)

- Сохраняется при переключении вкладок

  

### Блок: Общий прогресс

**Данные:**

Придумать и вставить самостоятельно

  

**Связи:**

- Учитывает только проверенные работы (score !== null)

- Пересчитывается при смене курса

  
  

### Блок: Календарь активности

**Функционал:**

- Отображает текущий месяц

- Дни с проверенными работами выделены

- Сегодняшний день отмечен

- Будущие дни приглушены

  

**Интерактивность:**

- Нажатие на день с активностью → открывает список работ за этот день

- Если одна работа — сразу открывает детали

- Если несколько — показывает модальное окно со списком

  

**Источник данных:** поле `timestamp` из вебхуков

  

---

  

## Страница: Статистика

Страница для визуализации прогресса ученика по каждому типу работ.

### Навигация по типам работ

**Кнопки-вкладки** для переключения между типами:
- Домашнее задание
- Пробник
- Эссе  
- Проект

При выборе типа показывается:
1. **График** с данными по этому типу работ
2. **Список последних работ** этого типа (2-3 последних, можно расширить до всех работ)

### Структура каждой вкладки

**Для каждого типа работ отображать:**
- График/визуализация результатов
- Средний балл (опционально)
- Последние 2-3 работы этого типа с названием, датой и оценкой

**Интерактивность:** 
- Нажатие на работу из списка → открывает детали
- Нажатие на элемент графика → открывает детали работы

### Данные для графиков

**Общие данные по каждому типу работ:**
- Список работ с оценками и датами
- Возможность рассчитать: средний балл, динамику, разницу между результатами

### График: Пробники (ВАЖНО)

**Данные:**
- Максимум: 100 баллов
- Обычно 3-5 пробников за курс

**Пример данных:**
```
Пробник №1: 72 балла (15 янв)
Пробник №2: 78 баллов (25 янв) → +6 от предыдущего
Пробник №3: 70 баллов (3 фев) → -8 от предыдущего
```

**ОБЯЗАТЕЛЬНО показывать:**
- Сам балл каждого пробника
- Разницу между соседними пробниками (+6, +7, -3 и т.д.)
- Визуальный индикатор роста/падения (стрелка, цвет, или другой способ)

**Возможные визуализации:** таймлайн с точками, горизонтальные столбцы, вертикальные столбцы с коннекторами — на твоё усмотрение.

### График: Эссе

**Данные:**
- Максимум: 50 баллов
- Обычно 3-7 эссе за курс

**Пример данных:**
```
Эссе: Реформы Александра II — 46/50 (3 фев)
Эссе: Причины революции 1917 — 42/50 (27 янв)
Эссе: Столыпинские реформы — 38/50 (20 янв)
```

**Тип графика:** столбчатая диаграмма

### График: Проекты

**Данные:**
- Максимум: 50 баллов
- Обычно 2-4 проекта за курс

**Пример данных:**
```
Проект: Первая мировая война — 47/50 (31 янв)
Проект: Индустриализация СССР — 43/50 (21 янв)
```

**Тип графика:** столбчатая диаграмма

### График: Домашние задания

**Данные:**
- Максимум: ПЕРЕМЕННЫЙ (зависит от задания: 10, 15, 25...)
- Много заданий (10-20 за курс)

**Пример данных:**
```
ДЗ: Анализ источника — 9/10 (4 фев)
ДЗ: Работа с картой — 8/10 (1 фев)
ДЗ: Крещение Руси — 20/25 (12 янв)
```

**Особенность:** Максимум разный у каждого задания, показывать просто балл (9/10, 20/25 и т.д.)

### Интерактивность

При нажатии на элемент графика — открывать детали работы (модальное окно)

## Страница: История

  

### Блок: Заголовок

- Название страницы

- Название текущего курса

  

### Блок: Список работ

**Данные для каждой работы:**

- Название работы

- Дата проверки 

- Оценка с максимумом 

  

**Сортировка:** по дате (новые сверху)

  

**Фильтрация:** только работы выбранного курса 

  

**Интерактивность:**

- Нажатие на карточку → открывает модальное окно с деталями

  

---

  

## Модальные окна

  

### Модальное окно: Детали работы

**Открывается из:**

- Страница История (нажатие на карточку работы)

- Страница Статистика (нажатие на точку пробника)

- Календарь активности (нажатие на день)

  

**Отображаемые данные:**

- Название работы

- Название курса

- Оценка с максимумом 

- Тип работы (Эссе / Проект / Пробник / Практикум)

- Дата проверки (или "Ожидает")

  

**Источник комментария:** поле `comment` из вебхука `lesson_task_accepted`

  

### Модальное окно: Работы за день

**Открывается из:** Календарь активности (если в этот день несколько работ)

  

**Отображаемые данные:**

- Дата (в формате "X месяца")

- Количество работ

- Список работ с:

- Названием

- Типом работы

- Оценкой

  

**Интерактивность:**

- Нажатие на работу → закрывает текущее окно и открывает детали работы

  

### Выпадающий список: Выбор курса

**Открывается из:** Блок выбора курса на главной

**Функционал:**
- Список всех курсов ученика
- Текущий курс выделен
- Выбор курса закрывает список и обновляет данные
